# Digital VLSI SoC Design and Planning

## Overview

Welcome to my documentation repository for the **Digital VLSI SoC Design and Planning** course. This repository chronicles my complete learning journey through the six-week hands-on program, covering the entire Digital ASIC Physical Design flow from **RTL to GDSII** using industry-relevant open-source EDA tools.

Unlike a conventional lab submission, this repository is designed as a comprehensive reference for students, beginners, and VLSI enthusiasts. Alongside laboratory exercises, it includes theoretical explanations, step-by-step procedures, command breakdowns, tool workflows, report analysis, troubleshooting, interview-oriented notes, and practical insights to help readers understand both **how** and **why** each stage of the design flow is performed.

## Why This Repository?

Modern VLSI engineers are expected to understand much more than Verilog or digital logic design. A complete understanding of the ASIC implementation flow—including synthesis, floorplanning, placement, clock tree synthesis (CTS), routing, static timing analysis (STA), power distribution network (PDN), and physical verification—is essential for building industry-ready skills.

The primary goal of this repository is to:

* Document every laboratory exercise in a structured and reproducible manner.
* Explain fundamental VLSI concepts before performing each experiment.
* Serve as a personal revision handbook for future projects and interviews.
* Provide a beginner-friendly learning resource for anyone starting their journey in Digital VLSI Physical Design.

## Course Objectives

Throughout this course, the following learning outcomes are achieved:

* Understand the complete RTL-to-GDSII ASIC implementation flow.
* Learn the purpose and significance of every stage in Digital Physical Design.
* Gain practical experience using GitHub Codespaces for cloud-based development.
* Perform synthesis, floorplanning, placement, CTS, routing, timing analysis, and PDN generation using open-source EDA tools.
* Analyze reports, optimize timing, and understand Engineering Change Orders (ECOs).
* Develop a professional documentation workflow for Digital VLSI projects.
