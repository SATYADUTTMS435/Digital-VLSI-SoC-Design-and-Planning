# Synthesis Output Files

After synthesis, OpenLANE generates the synthesized design files that are used as inputs for the subsequent stages of the RTL-to-GDSII flow.

---

## 1. picorv32a.v

### Purpose

This is the synthesized gate-level Verilog netlist generated by the synthesis tool.

### What does it contain?

- Standard cell instantiations
- Logic connections
- Optimized gate-level implementation
- Technology-mapped design

### Why is it important?

Unlike the original RTL, this file represents the actual hardware implementation using SKY130 standard cells. It serves as the primary input for floorplanning, placement, clock tree synthesis, routing, and timing analysis.

---

## 2. picorv32a.sdf

### Purpose

This is the Standard Delay Format (SDF) file generated after synthesis.

### What does it contain?

- Cell propagation delays
- Timing arcs
- Delay annotations
- Timing information for simulation

### Why is it important?

The SDF file is used during gate-level simulation and static timing analysis to model realistic propagation delays, enabling accurate timing verification before fabrication.

---

## Summary

The synthesized Verilog netlist and the SDF file together form the primary outputs of the synthesis stage. These files are passed to the physical design flow, where the logical design is transformed into a manufacturable integrated circuit layout.
