## Generated Design Files

During the physical design and characterization of the custom CMOS inverter, several files are generated at different stages of the design flow. Each file serves a specific purpose, ranging from storing the layout geometry to enabling circuit simulation and integration into the ASIC design flow.

### `sky130_inv.mag`
The **`.mag`** file is the native layout database of the Magic VLSI Layout Editor. It stores the complete physical layout of the CMOS inverter, including all fabrication layers, device geometries, ports, labels, and technology-specific properties. This file is used for layout editing, Design Rule Checking (DRC), layout extraction, and LEF generation.

### `sky130_inv.ext`
The **`.ext`** file is the extracted layout representation generated by Magic after running the extraction process. It contains connectivity information and extracted parasitic elements such as resistances and capacitances, serving as an intermediate file for generating the SPICE netlist used in circuit simulation.

### `sky130_inv.spice`
The **`.spice`** file is the extracted SPICE netlist of the CMOS inverter. It represents the circuit using transistor models, node connections, and extracted parasitic components, allowing accurate electrical simulation in ngspice for timing, functionality, and performance characterization.

### `sky130_vsdinv.lef`
The **`.lef`** (Library Exchange Format) file provides an abstract physical description of the standard cell, including its dimensions, pin locations, routing blockages, and placement information. Unlike the complete layout, the LEF file contains only the information required by place-and-route tools, enabling the custom inverter to be integrated into the OpenLane ASIC design flow as a standard cell.
